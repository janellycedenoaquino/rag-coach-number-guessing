# Portfolio Reflection — What This Project Says About Me as an AI Engineer

Building **RAG Coach: Number Guessing** showed me that I treat the LLM as the smallest part of the system, not the centerpiece. The model call in [`ai_coach.py`](ai_coach.py) is one line; everything around it — rule-based retrieval over a 5-doc knowledge base, input redaction, a ±2 output guardrail with structured logging, a draft-→-critique-→-regenerate agent loop, a 12-scenario eval harness, and a baseline-vs-constrained A/B that quantifies what the prompt is actually buying — is where the engineering lives. The clearest proof of that mindset is the ±2 buffer in [`validate_response`](guardrails.py): my first version only blocked the literal secret, until `llama3.2` told me to "try 43" with secret 42 and I realized the leak surface was wider than the equality check. I wrote the regression test the same day, added it to the harness, and documented why the rule exists in [`model_card.md`](model_card.md). I made deliberate "no" calls too — no vector store for five short docs, no self-rated confidence score for a model whose calibration I don't trust — and recorded the reasoning so the absences are decisions, not omissions. What this project says about me is that I care more about being able to defend every layer of a system than about adding layers for their own sake: I'd rather ship a smaller system I can fully explain, test, and prove safe than a larger one whose behavior I'd have to hand-wave.
